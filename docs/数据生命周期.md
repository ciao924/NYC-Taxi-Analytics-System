## 二、分层详细策略

### 2.1 ODS 层生命周期

**表名**：`taxi_trip_green_ods`, `taxi_trip_yellow_ods`

| 阶段 | 保留周期 | 存储 | 操作 |
|------|----------|------|------|
| 热数据 | 0-7 天 | HDFS / SSD | 全量访问 |
| 温数据 | 8-30 天 | HDFS | 低频访问 |
| 冷数据 | 31-90 天 | HDFS | 归档 |
| 归档 | 90 天后 | 压缩文件 | 可恢复 |

**Hive TTL 配置**：

```sql
-- 设置分区过期时间（90天）
ALTER TABLE nyc_taxi_ods.taxi_trip_green_ods SET TBLPROPERTIES (
  'parquet.compression'='ZSTD',
  'expire_partitionTTL'='90d'
);
```

**清理脚本**：

```bash
#!/bin/bash
# ods_data_cleanup.sh

# ODS 数据保留 90 天
RETENTION_DAYS=90
CUTOFF_DATE=$(date -d "-${RETENTION_DAYS} days" +%Y-%m-%d)

echo "清理 ODS 分区，早于 ${CUTOFF_DATE} 的分区将被删除"

# 清理绿表
hdfs dfs -ls /user/hive_local/warehouse/nyc_taxi_ods.db/taxi_trip_green_ods/dt=* | \
  while read line; do
    PARTITION_DATE=$(echo $line | awk '{print $NF}' | grep -oP '\d{4}-\d{2}-\d{2}')
    if [[ "$PARTITION_DATE" < "$CUTOFF_DATE" ]]; then
      echo "删除分区: taxi_trip_green_ods/dt=$PARTITION_DATE"
      hdfs dfs -rm -r "/user/hive_local/warehouse/nyc_taxi_ods.db/taxi_trip_green_ods/dt=$PARTITION_DATE"
    fi
  done

# 清理黄表
hdfs dfs -ls /user/hive_local/warehouse/nyc_taxi_ods.db/taxi_trip_yellow_ods/dt=* | \
  while read line; do
    PARTITION_DATE=$(echo $line | awk '{print $NF}' | grep -oP '\d{4}-\d{2}-\d{2}')
    if [[ "$PARTITION_DATE" < "$CUTOFF_DATE" ]]; then
      echo "删除分区: taxi_trip_yellow_ods/dt=$PARTITION_DATE"
      hdfs dfs -rm -r "/user/hive_local/warehouse/nyc_taxi_ods.db/taxi_trip_yellow_ods/dt=$PARTITION_DATE"
    fi
  done

echo "ODS 清理完成"
```

**MySQL 分区清理**：

```sql
-- 创建按月分区的 ADS 表
CREATE TABLE analysis_kpi_daily (
  id BIGINT NOT NULL AUTO_INCREMENT,
  stat_date DATE NOT NULL,
  total_trips BIGINT,
  total_revenue DECIMAL(15,2),
  avg_fare DECIMAL(10,2),
  PRIMARY KEY (id, stat_date)
) PARTITION BY RANGE (YEAR(stat_date) * 100 + MONTH(stat_date)) (
  PARTITION p202501 VALUES LESS THAN (202502),
  PARTITION p202502 VALUES LESS THAN (202503),
  PARTITION p202503 VALUES LESS THAN (202504),
  PARTITION p202504 VALUES LESS THAN (202505),
  PARTITION p_future VALUES LESS THAN MAXVALUE
);

-- 清理超过 365 天的数据
DELETE FROM analysis_kpi_daily
WHERE stat_date < DATE_SUB(CURDATE(), INTERVAL 365 DAY);
```

## 四、小文件合并策略

### 4.1 小文件合并规则

| 层 | 文件大小阈值 | 合并目标大小 | 触发时机 |
|----|-------------|-------------|----------|
| ODS | < 128MB | 256MB | 每日调度后 |
| DWD | < 128MB | 256MB | 每日调度后 |
| DWS | < 128MB | 256MB | 每日调度后 |

### 4.2 小文件合并脚本

```scala
// SmallFileMergeTask.scala

package com.taxi.etl.common

import org.apache.spark.sql.SparkSession

object SmallFileMergeTask {

  def main(args: Array[String]): Unit = {
    val spark = SparkSession.builder()
      .enableHiveSupport()
      .getOrCreate()

    val tableName = args(0)
    val targetFileSize = args.lift(1).getOrElse("256MB").toString
    val partitionSpec = args.lift(2).getOrElse("")

    try {
      val tablePath = tableName match {
        case "ods_green" => "/user/hive_local/warehouse/nyc_taxi_ods.db/taxi_trip_green_ods"
        case "ods_yellow" => "/user/hive_local/warehouse/nyc_taxi_ods.db/taxi_trip_yellow_ods"
        case "dwd" => "/user/hive_local/warehouse/nyc_taxi_dwd.db/dwd_taxi_trip_detail"
        case "dws" => "/user/hive_local/warehouse/nyc_taxi_dws.db/dws_taxi_daily_stats"
        case _ => throw new IllegalArgumentException(s"Unknown table: $tableName")
      }

      // 读取数据并重新写入，合并小文件
      val df = spark.read.format("parquet").load(tablePath)

      df.repartition(df.rdd.getNumPartitions / 2)
        .write
        .format("parquet")
        .option("parquet.compression", "ZSTD")
        .mode("overwrite")
        .partitionBy("dt")
        .save(tablePath)

      println(s"小文件合并完成: $tableName")

    } finally {
      spark.stop()
    }
  }
}
```

## 五、备份与恢复策略

### 5.1 备份策略

| 数据类型 | 备份频率 | 保留周期 | 存储位置 |
|----------|----------|----------|----------|
| Hive 元数据 | 每日 | 30 天 | 备份服务器 |
| ADS MySQL | 每日 | 90 天 | 备份服务器 |
| Kafka 消费点位 | 每小时 | 7 天 | Zookeeper |
| Flink Checkpoint | 每 10 分钟 | 3 个 | HDFS |

### 5.2 备份脚本

```bash
#!/bin/bash
# backup_ads_mysql.sh

BACKUP_DIR=/data/backup/mysql
DATE=$(date +%Y%m%d)
RETENTION_DAYS=90

# 创建备份目录
mkdir -p ${BACKUP_DIR}/${DATE}

# 备份 ADS 数据库
mysqldump -h 192.168.127.102 -u root -p'BAi@123456' \
  --single-transaction \
  --routines \
  --triggers \
  nyc_taxi_ads | gzip > ${BACKUP_DIR}/${DATE}/nyc_taxi_ads.sql.gz

# 删除过期备份
find ${BACKUP_DIR} -type f -mtime +${RETENTION_DAYS} -delete

echo "MySQL 备份完成: ${DATE}"
```

## 六、监控与告警

### 6.1 生命周期监控指标

| 指标 | 告警阈值 | 说明 |
|------|----------|------|
| ODS 分区数量 | > 100 | 分区数量异常 |
| ODS 数据量增长率 | > 50% | 数据量增长异常 |
| 小文件数量 | > 1000 | 小文件过多 |
| Checkpoint 失败次数 | >= 1 | Checkpoint 失败 |
| Kafka Lag | > 10000 | 消费延迟过大 |
| 磁盘使用率 | > 80% | 磁盘空间不足 |

### 6.2 监控脚本

```bash
#!/bin/bash
# check_partition_count.sh

TABLE_NAME=$1
MAX_PARTITIONS=100

PARTITION_COUNT=$(hdfs dfs -count /user/hive_local/warehouse/nyc_taxi_ods.db/${TABLE_NAME} 2>/dev/null | awk '{print $2}')

if [ $PARTITION_COUNT -gt $MAX_PARTITIONS ]; then
  echo "告警: ${TABLE_NAME} 分区数量 ${PARTITION_COUNT} 超过阈值 ${MAX_PARTITIONS}"
  # 发送告警到钉钉
  curl -X POST "https://oapi.dingtalk.com/robot/send" \
    -H "Content-Type: application/json" \
    -d '{"msgtype": "text", "text": {"content": "[告警] ${TABLE_NAME} 分区数量超限"}}'
fi
```

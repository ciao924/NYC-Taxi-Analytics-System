
# 纽约市出租车数据分析系统 - 完整项目分析报告

## 1. 项目概述

### 1.1 项目定位

**纽约市出租车数据分析系统**（NYC Taxi Analytics System）是一个综合性的**实时 + 离线**数据仓库与分析平台，通过对纽约市 Yellow Taxi 和 Green Taxi 数据的处理、分析和可视化，为业务决策提供数据支撑。

### 1.2 核心目标

| 目标 | 描述 | 实现状态 |
|------|------|----------|
| **实时数据处理** | 消费 Kafka 实时数据，完成 ODS 落地、指标计算、质量检测 | ✅ |
| **离线数据仓库** | 构建 ODS -&gt; DWD -&gt; DWS -&gt; ADS 完整分层数仓 | ✅ |
| **数据质量保障** | 全链路数据质量监控、异常检测、告警推送 | ✅ |
| **智能查询分析** | 支持自然语言查询、SQL 生成、图表可视化 | ✅ |
| **数据可视化** | 提供多维度图表、热力图、实时指标展示 | ✅ |

## 3. 模块详解

### 3.1 data-producer (数据生产模块)

| 属性 | 说明 |
|------|------|
| **技术栈** | Python + Kafka Producer |
| **核心功能** | 读取 Parquet 源数据，发送至 Kafka 消息队列 |
| **主要特性** | 速率控制、断点续传、配置灵活 |
| **文档位置** | `data-producer/docs/MODULE_ANALYSIS.md` |

**关键实现**：
- 速率控制：通过 `rate_limit` 配置每秒发送记录数
- 断点续传：使用 Checkpoint 文件记录已发送的偏移量
- Kafka 主题：`nyc-yellow-trips`、`nyc-green-trips`

---

### 3.2 streaming-processor (实时处理模块)

| 属性 | 说明 |
|------|------|
| **技术栈** | Flink + Kafka + HDFS + MySQL |
| **核心功能** | 实时 ODS 落地、实时指标计算、质量检测与告警 |
| **架构设计** | 两个独立 Job，解耦实时数仓和实时指标 |
| **文档位置** | `streaming-processor/docs/MODULE_ANALYSIS.md` |

**核心 Job**：
1. **RealtimeOdsJob**：实时 ODS 数据落地
   - 消费 Kafka → 数据清洗 → 写 HDFS (ORC)
   - 支持 Checkpoint 和状态恢复

2. **RealtimeMetricsJob**：实时指标计算
   - 5 分钟滑动窗口聚合订单指标
   - 热门区域 TOP N 统计
   - 费用构成统计
   - 数据质量检测与告警推送

---

### 3.3 batch-processor (离线处理模块)

| 属性 | 说明 |
|------|------|
| **技术栈** | Spark + Hive + MySQL |
| **核心功能** | 构建 ODS -&gt; DWD -&gt; DWS -&gt; ADS 分层数仓 |
| **数仓分层** | ODS（原始）→ DWD（明细）→ DWS（汇总）→ ADS（应用） |
| **文档位置** | `batch-processor/docs/MODULE_ANALYSIS.md` |

**核心构建器**：
- **DwdLayerBuilder**：明细层构建
  - 数据清洗、空值填充、异常值过滤
  - 维度表关联（Zone、Borough 等）
  - 数据质量检查

- **DwsLayerBuilder**：汇总层构建
  - 按日、小时、区域等维度聚合
  - 热门区域、费用构成等指标

- **AdsLayerBuilder**：应用层构建
  - 面向业务的指标表
  - 写入 MySQL 供查询

## 4. 数仓分层设计

### 4.1 分层架构

| 层级 | 全称 | 定位 | 数据来源 | 刷新频率 |
|------|------|------|----------|----------|
| **ODS** | Operational Data Store | 原始数据层 | 源数据 (Parquet) | 批量/实时 |
| **DWD** | Data Warehouse Detail | 明细数据层 | ODS | 每日批量 |
| **DWS** | Data Warehouse Service | 汇总数据层 | DWD | 每日批量 |
| **ADS** | Application Data Store | 应用数据层 | DWS | 每日批量 |

### 4.2 命名规范

| 层级 | 前缀规范 | 示例 |
|------|----------|------|
| ODS | `ods_` | `ods_yellow_taxi_tripdata` |
| DWD | `dwd_` | `dwd_yellow_taxi_trip_fact` |
| DWS | `dws_` | `dws_taxi_trip_hourly_agg` |
| ADS | `ads_` | `ads_taxi_kpi_daily` |

## 5. 技术选型分析

### 5.1 大数据技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| **消息队列** | Kafka | 高吞吐、高可用、生态成熟 |
| **实时计算** | Flink | 低延迟、状态管理、Exactly-Once |
| **离线计算** | Spark | 高性能、SQL 支持好、生态丰富 |
| **数据仓库** | Hive | 分区管理、元数据管理 |
| **数据仓库** | Hive | SQL 接口、与 Spark/Flink 集成好 |
| **文件存储** | HDFS | 高可靠、高吞吐、成本低 |

### 5.2 应用技术栈

## 6. 核心亮点

### 6.1 AI 智能查询系统

本系统的核心亮点是 **AI 智能对话查询功能**，支持：

1. **自然语言输入**：用户用中文提问，如 "2025年1月的收入趋势"
2. **SQL 自动生成**：LLM 理解意图，自动生成 SQL 查询
3. **图表智能推荐**：根据数据特征，推荐最佳可视化方式
4. **会话管理**：支持多轮对话，历史会话持久化

**技术实现**：
- 支持 Coze 和 DeepSeek 两种大模型
- DSL 中间层，提高 SQL 生成准确率
- SQL 安全检查，防止注入攻击
- Schema 自动检索，无需表结构知识

### 6.2 实时 + 离线双轨数据处理

| 维度 | 实时处理 (Flink) | 离线处理 (Spark) |
|------|------------------|------------------|
| **延迟** | 秒级/分钟级 | T+1 |
| **用途** | 实时监控、告警、运营大屏 | 深度分析、报表、历史趋势 |
| **特点** | 增量、状态化 | 全量、批处理 |

### 6.3 全链路数据质量保障

1. **实时质量检测** (Flink)
   - 空值检测
   - 值域检测
   - 时间合理性检测
   - 异常告警推送

2. **离线质量检查** (Spark)
   - 源数据记录数校验
   - 一致性验证
   - 质量报告生成

### 6.4 前后端类型统一

通过 `@taxi-analytics/shared-types` 包，确保前后端数据类型完全一致，减少对接错误。

## 8. 快速启动指南

### 8.1 环境要求

- Java 17+
- Python 3.9+
- Node.js 18+
- Maven 3.9+
- Hadoop 3.x (HDFS)
- Hive 3.x
- Kafka 2.8+
- MySQL 8.0+

### 8.2 启动顺序

```bash
# 1. 启动数据生产
cd data-producer
python scripts/kafka_producer.py

# 2. 启动 Flink 实时处理 (可选)
cd streaming-processor
flink run -c com.taxi.realtime.RealtimeOdsJob target/streaming-processor-1.0.jar
flink run -c com.taxi.realtime.RealtimeMetricsJob target/streaming-processor-1.0.jar

# 3. 启动 Spark 离线处理
cd batch-processor
spark-submit --class com.taxi.etl.dwd.DwdLayerBuilder target/batch-processor-1.0.jar
spark-submit --class com.taxi.etl.dws.DwsLayerBuilder target/batch-processor-1.0.jar
spark-submit --class com.taxi.etl.ads.AdsLayerBuilder target/batch-processor-1.0.jar

# 4. 启动后端服务
cd taxi/taxi-analytics-backend
mvn spring-boot:run

# 5. 启动前端应用
cd taxi/taxi-analytics-frontend
npm install
npm run dev
```

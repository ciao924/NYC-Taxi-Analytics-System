## 一、告警概述

### 1.1 告警方式

| 告警方式 | 状态 | 说明 |
|----------|------|------|
| 钉钉 | 启用 | 主要告警方式 |
| 邮件 | 可选 | 备用告警方式 |
| 短信 | 可选 | 紧急告警 |

### 1.2 告警类型

| 告警类型 | 触发条件 | 告警级别 |
|----------|----------|----------|
| 工作流失败 | 工作流执行失败 | 严重 |
| 任务超时 | 任务执行超过阈值 | 警告 |
| 数据质量 | 质量评分低于阈值 | 警告 |
| Kafka Lag | 消费延迟超过阈值 | 警告 |
| 系统异常 | 系统资源不足 | 严重 |

## 二、钉钉告警配置

### 2.1 钉钉机器人配置

**步骤 1**：在钉钉群中添加自定义机器人

1. 打开钉钉群 → 设置 → 智能群助手
2. 添加机器人 → 自定义
3. 机器人名字：`TaxiDataAlert`
4. 安全设置：选择"自定义关键词"
5. 关键词：`告警` 或 `Taxi`

**步骤 2**：获取 Webhook 地址

```
https://oapi.dingtalk.com/robot/send?access_token=XXXXXXXXXXXXXXXXX
```

### 2.2 DolphinScheduler 钉钉告警配置

**配置文件**：`dolphinscheduler-3.1.8/alert-server/conf/application.yaml`

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://192.168.127.102:3306/dolphinscheduler?useUnicode=true&characterEncoding=utf8
    username: dsuser
    password: Ds@123456

alert:
  # 告警类型
  alert-type:
    - DINGTALK
  # 钉钉配置
  dingtalk:
    webhook-url: https://oapi.dingtalk.com/robot/send?access_token=XXXXXXXXXXXXXXXXX
    msgtype: MARKDOWN
    # 关键词列表（必须包含）
    keyword: 告警
    # 是否启用代理
    enable-proxy: false
```

### 3.4 Kafka Lag 告警

| 配置项 | 值 |
|--------|-----|
| 告警名称 | kafka_lag_alert |
| 告警类型 | Kafka 消费延迟 |
| 触发条件 | Lag > 10000 |
| 告警方式 | 钉钉 |
| 检查频率 | 每 5 分钟 |

**Kafka Lag 监控脚本**：

```bash
#!/bin/bash
# kafka_lag_monitor.sh

KAFKA_HOME=/opt/module/kafka_2.13
KAFKA_TOPICS="taxi_trip_green taxi_trip_yellow"
LAG_THRESHOLD=10000
DINGTALK_WEBHOOK="https://oapi.dingtalk.com/robot/send?access_token=XXX"

check_lag() {
  local topic=$1
  local lag=$( ${KAFKA_HOME}/bin/kafka-consumer-groups.sh \
    --bootstrap-server hadoop102:9092,hadoop103:9092,hadoop104:9092 \
    --group flink-consumer-group \
    --topic ${topic} \
    --describe 2>/dev/null | grep LAG | awk '{sum+=$NF} END {print sum}')

  if [ ${lag} -gt ${LAG_THRESHOLD} ]; then
    send_alert "Kafka Lag 告警" "${topic} 消费延迟 ${lag} 超过阈值 ${LAG_THRESHOLD}"
  fi
}

send_alert() {
  local title=$1
  local content=$2
  curl -X POST ${DINGTALK_WEBHOOK} \
    -H "Content-Type: application/json" \
    -d "{
      'msgtype': 'markdown',
      'markdown': {
        'title': '${title}',
        'text': '## 🚨 ${title}\n\n${content}'
      }
    }"
}

for topic in ${KAFKA_TOPICS}; do
  check_lag ${topic}
done
```

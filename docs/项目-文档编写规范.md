## 3. Markdown 编写规范

### 3.1 标题层级

```markdown
# 一级标题（H1）
## 二级标题（H2）
### 三级标题（H3）
#### 四级标题（H4）
```

### 3.2 列表

```markdown
- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2
```

### 3.3 代码块

```markdown
```语言
代码内容
```
```

### 3.4 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

### 3.5 链接

```markdown
[显示文本](链接地址)

[相对路径文档](../another-module/docs/README.md)
```

## 5. 文档维护指南

### 5.1 新增模块时

1. ✅ 在模块下创建 `docs/` 目录
2. ✅ 编写 `README.md`（使用说明）
3. ✅ 编写 `MODULE_ANALYSIS.md`（分析报告）
4. ✅ 在项目 `docs/DOCUMENTATION_GUIDELINES.md` 的索引中添加新模块
5. ✅ 在项目 `docs/COMPLETE_PROJECT_ANALYSIS_REPORT.md` 中更新模块说明

### 5.2 修改代码时

1. ✅ 如果修改影响功能，更新对应模块的 `README.md`
2. ✅ 如果修改影响架构，更新对应模块的 `MODULE_ANALYSIS.md`
3. ✅ 在 `CHANGELOG.md` 中记录变更内容

### 5.3 版本发布时

1. ✅ 更新所有变更记录
2. ✅ 确保文档与代码一致
3. ✅ 更新版本号和日期

## 6. 文档格式检查清单

在提交文档前，请确认：

- [ ] 标题层级正确（H1 → H2 → H3）
- [ ] 表格对齐美观
- [ ] 代码块语言标识正确
- [ ] 链接路径正确（相对路径 vs 绝对路径）
- [ ] 无拼写错误
- [ ] 末尾有文档版本和生成日期
- [ ] 中英文之间有空格（如：`Java 技术` 而非 `Java技术`）

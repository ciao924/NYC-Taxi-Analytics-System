# 出租车大数据分析系统 - 数仓命名规范

**文档版本**：v1.0
**创建日期**：2026-04-26
**适用版本**：v4.0+

---

## 一、数据库命名规范

### 1.1 命名原则

| 原则 | 说明 |
|------|------|
| 业务导向 | 名称应反映业务领域 |
| 层次分明 | 通过命名体现数据层次 |
| 简洁清晰 | 使用通用缩写，避免歧义 |
| 统一规范 | 全部小写，单词间用下划线分隔 |

### 1.2 数据库命名规则

```
[业务前缀]_[层次]_[用途修饰]
```

### 1.3 层级数据库定义

| 数据库名 | 层次 | 用途 | 存储格式 |
|----------|------|------|----------|
| `nyc_taxi_ods` | ODS | 原始数据层 | ORC |
| `nyc_taxi_dwd` | DWD | 明细数据层 | Iceberg |
| `nyc_taxi_dws` | DWS | 汇总数据层 | Iceberg |
| `nyc_taxi_dim` | DIM | 维度数据层 | ORC |
| `nyc_taxi_ads` | ADS | 应用数据层 | MySQL |
| `nyc_taxi_dead_letter` | DL | 死信数据层 | ORC |
| `nyc_taxi_realtime` | RT | 实时数据层 | MySQL |

### 1.4 特殊数据库

| 数据库名 | 用途 |
|----------|------|
| `dolphinscheduler` | DolphinScheduler 元数据库 |
| `spark_history` | Spark 历史日志存储 |

## 二、表命名规范

### 2.1 通用命名规则

```
[层标识]_[业务主题]_[时间范围]_[更新类型]_[分区标识]
```

**命名元素说明**：

| 元素 | 说明 | 示例 |
|------|------|------|
| 层标识 | 数据层次 | ods/dwd/dws/ads |
| 业务主题 | 业务领域 | taxi_trip / passenger / location |
| 时间范围 | 数据时间粒度 | daily/hourly/monthly |
| 更新类型 | 全量/增量 | full/incremental |
| 分区标识 | 特殊分区说明 | partition/range |

### 2.2 ODS 层表命名

```
ods_[数据来源]_[原始表名]
```

**示例**：

| 表名 | 说明 |
|------|------|
| `ods_taxi_green_trip` | 绿表原始行程数据 |
| `ods_taxi_yellow_trip` | 黄表原始行程数据 |
| `ods_kafka_taxi_trip_green` | Kafka 接入的绿表数据 |

### 2.3 DWD 层表命名

```
dwd_[业务主题]_[事实类型]_[修饰词]
```

**示例**：

| 表名 | 说明 |
|------|------|
| `dwd_taxi_trip_detail` | 行程明细表 |
| `dwd_taxi_trip_valid` | 有效行程表 |
| `dwd_taxi_passenger_detail` | 乘客明细表 |
| `dwd_dim_date` | 日期维度表 |
| `dwd_dim_location` | 位置维度表 |

### 2.4 DWS 层表命名

```
dws_[业务主题]_[聚合粒度]_[时间周期]
```

**示例**：

| 表名 | 说明 |
|------|------|
| `dws_taxi_hourly_stats` | 每小时行程统计 |
| `dws_taxi_daily_stats` | 每日行程统计 |
| `dws_taxi_zone_flow` | 区域人流量统计 |

### 2.5 ADS 层表命名

```
ads_[业务指标类型]_[时间范围]_[更新频率]
```

**示例**：

| 表名 | 说明 |
|------|------|
| `analysis_kpi_daily` | 每日 KPI 指标 |
| `analysis_hourly_distribution` | 每小时分布统计 |
| `analysis_pickup_hotspots` | 上车热点区域 |
| `analysis_fee_composition` | 费用构成分析 |

### 2.6 维度表命名

```
dim_[维度主题]_[类型标识]
```

**示例**：

| 表名 | 说明 |
|------|------|
| `dim_date` | 日期维度 |
| `dim_location_zone` | 区域维度 |
| `dim_payment_type` | 支付类型维度 |
| `dim_vendor` | 供应商维度 |
| `dim_airport` | 机场维度 |

### 2.7 实时表命名

```
realtime_[业务主题]_[粒度]
```

**示例**：

| 表名 | 说明 |
|------|------|
| `realtime_trip_green` | 绿表实时数据 |
| `realtime_kpi_5min` | 5分钟实时KPI |

## 三、字段命名规范

### 3.1 通用字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | BIGINT | 主键 |
| `stat_date` | STRING | 统计日期 (yyyy-MM-dd) |
| `stat_hour` | INT | 统计小时 (0-23) |
| `hour_of_day` | INT | 小时标识 (0-23) |
| `day_of_week` | INT | 星期几 (1-7) |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |
| `etl_time` | TIMESTAMP | ETL 处理时间 |
| `source_system` | STRING | 数据来源系统 |

### 3.2 ODS 层字段

保持与原始数据一致，使用原始字段名。

### 3.3 DWD 层字段

| 前缀 | 说明 | 示例 |
|------|------|------|
| `pu_` | 上车相关 | pu_location_id, pu_borough |
| `do_` | 下车相关 | do_location_id, do_borough |
| `trip_` | 行程相关 | trip_distance, trip_duration |
| `fare_` | 费用相关 | fare_amount, fare_tip |
| `time_` | 时间相关 | time_pickup, time_dropoff |

### 3.4 ADS 层字段

| 字段类型 | 命名规则 | 示例 |
|----------|----------|------|
| 计数类 | `xxx_count` | trip_count, passenger_count |
| 金额类 | `xxx_amount` | total_amount, avg_amount |
| 比率类 | `xxx_ratio` | trip_ratio, revenue_ratio |
| 平均类 | `xxx_avg` | avg_fare, avg_distance |
| 极值类 | `xxx_max/min` | max_fare, min_distance |
| 合计类 | `xxx_total` | total_trips, total_revenue |

### 3.5 字段命名禁忌

| 禁忌 | 正确做法 |
|------|----------|
| 使用拼音 | 使用英文单词 |
| 使用缩写 | 使用完整单词或公认缩写 |
| 数字开头 | 使用字母开头 |
| 特殊字符 | 仅使用下划线分隔 |
| 与 SQL 关键字冲突 | 添加前缀或改用其他词 |

## 八、存储格式规范

### 8.1 分层存储格式

| 层次 | 存储格式 | 压缩方式 | 说明 |
|------|----------|----------|------|
| ODS | ORC | ZSTD (level 6) | 原始数据，保留全量 |
| DWD | Iceberg | ZSTD (level 6) | 明细数据，支持 ACID |
| DWS | Iceberg | ZSTD (level 6) | 汇总数据，支持 Time Travel |
| DIM | ORC | ZSTD (level 6) | 维度数据 |
| ADS | MySQL | - | 应用数据，直接查询 |

### 8.2 文件大小要求

| 层 | 最小文件大小 | 最大文件大小 | 目标文件数/分区 |
|----|-------------|-------------|----------------|
| ODS | 128MB | 256MB | 1-10 |
| DWD | 128MB | 256MB | 1-10 |
| DWS | 128MB | 256MB | 1-5 |

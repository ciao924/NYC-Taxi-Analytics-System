## 二、ODS 层职责

### 2.1 职责定义

| 职责项 | 说明 |
|--------|------|
| 数据接入 | 从 Kafka、Parquet 文件等数据源接入原始数据 |
| 数据原貌保持 | 不做任何业务加工，保持数据原样 |
| 分区管理 | 按时间(tz)和数据类型(taxi_type)分区存储 |
| 数据校验 | 记录数对账、格式校验 |
| 死信处理 | 异常数据写入 dead_letter 表 |

### 2.2 数据存储

| 配置项 | 值 |
|--------|-----|
| 数据库 | nyc_taxi_ods |
| 存储格式 | ORC |
| 压缩方式 | ZSTD (level 6) |
| 分区字段 | dt, taxi_type |
| HDFS 路径 | `/user/hive_local/warehouse/nyc_taxi_ods.db/` |

### 2.3 ODS 表清单

| 表名 | 数据来源 | 说明 |
|------|----------|------|
| `taxi_trip_green_ods` | Parquet / Kafka | 绿表原始行程数据 |
| `taxi_trip_yellow_ods` | Parquet / Kafka | 黄表原始行程数据 |

### 2.4 ODS 层边界

**上游**：
- Parquet 文件（py_taxi/data/）
- Kafka Topic（taxi_trip_green, taxi_trip_yellow）

**下游**：
- DWD 层（通过 Spark ETL 消费 ODS 数据）

## 三、DWD 层职责

### 3.1 职责定义

| 职责项 | 说明 |
|--------|------|
| 数据清洗 | 空值处理、异常值过滤、数据类型转换 |
| 数据标准化 | 字段命名统一、编码标准化 |
| 维度关联 | 与维度表关联，补充维度信息 |
| SCD Type2 | 支持缓慢变化维度 |
| 数据隔离 | 清洗后的干净数据与原始数据分离 |

### 3.2 数据存储

| 配置项 | 值 |
|--------|-----|
| 数据库 | nyc_taxi_dwd |
| 存储格式 | Iceberg |
| 压缩方式 | ZSTD (level 6) |
| 分区字段 | dt |
| 表类型 | Iceberg (支持 ACID 和 Time Travel) |

### 3.3 DWD 表清单

| 表名 | 说明 |
|------|------|
| `dwd_taxi_trip_detail` | 行程明细表 |
| `dwd_taxi_trip_valid` | 有效行程表 |
| `dwd_dim_date` | 日期维度表 |
| `dwd_dim_location` | 位置维度表 |
| `dwd_dim_payment_type` | 支付类型维度表 |
| `dwd_dim_vendor` | 供应商维度表 |
| `dwd_dim_airport` | 机场维度表 |

### 3.4 DWD 层数据清洗规则

| 规则类型 | 规则说明 |
|----------|----------|
| 空值处理 | passenger_count=0 设置为 null |
| 范围校验 | trip_distance > 0 AND < 500 |
| 时间校验 | tpep_pickup_datetime < tpep_dropoff_datetime |
| 金额校验 | fare_amount >= 0 AND total_amount >= fare_amount |
| 坐标校验 | pickup_longitude BETWEEN -180 AND 180 |

### 3.5 DWD 层边界

**上游**：
- ODS 层（taxi_trip_green_ods, taxi_trip_yellow_ods）

**下游**：
- DWS 层（按日期分区关联聚合）
- DIM 层（维度表作为广播维度）

## 四、DWS 层职责

### 4.1 职责定义

| 职责项 | 说明 |
|--------|------|
| 数据汇总 | 按业务主题进行轻度汇总 |
| 主题建模 | 按行程、费用、区域等主题组织数据 |
| 粒度统一 | 保证汇总粒度一致性 |
| 指标计算 | 计算派生指标和聚合指标 |

### 4.2 数据存储

| 配置项 | 值 |
|--------|-----|
| 数据库 | nyc_taxi_dws |
| 存储格式 | Iceberg |
| 压缩方式 | ZSTD (level 6) |
| 分区字段 | dt |
| 表类型 | Iceberg |

### 4.3 DWS 表清单

| 表名 | 聚合粒度 | 说明 |
|------|----------|------|
| `dws_taxi_hourly_stats` | 小时 | 每小时行程统计 |
| `dws_taxi_daily_stats` | 天 | 每日行程统计 |
| `dws_taxi_zone_flow` | 天+区域 | 区域人流量统计 |

### 4.4 DWS 层指标定义

| 指标名 | 计算方式 | 说明 |
|--------|----------|------|
| `trip_count` | COUNT(*) | 行程数 |
| `total_revenue` | SUM(fare_amount + extra + mta_tax + tip + tolls + improvement_surcharge) | 总收入 |
| `avg_fare` | AVG(fare_amount) | 平均票价 |
| `avg_distance` | AVG(trip_distance) | 平均距离 |
| `avg_duration` | AVG(dropoff_datetime - pickup_datetime) | 平均时长 |

### 4.5 DWS 层边界

**上游**：
- DWD 层（明细数据）

**下游**：
- ADS 层（读取 DWS 数据生成应用指标）

## 七、实时层架构

### 7.1 Lambda 架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         数据源                                  │
│              (Parquet → Python → Kafka)                        │
└─────────────────────────────┬───────────────────────────────────┘
                              │
        ┌─────────────────────┴─────────────────────┐
        │                                           │
        ▼                                           ▼
┌───────────────────┐                     ┌───────────────────┐
│    Speed Layer    │                     │    Batch Layer    │
│   (Flink 实时)     │                     │  (Spark 离线)     │
│                   │                     │                   │
│ Kafka → Flink →   │                     │ ODS → DWD → DWS   │
│ MySQL Realtime    │                     │                   │
└─────────┬─────────┘                     └─────────┬─────────┘
          │                                       │
          └───────────────────┬───────────────────┘
                              │
                              ▼
                    ┌───────────────────┐
                    │   Serving Layer    │
                    │   (应用服务层)     │
                    │                   │
                    │ Spring Boot API   │
                    │ Vue 3 Dashboard   │
                    └───────────────────┘
```

### 7.2 实时层职责

| 层次 | 职责 | 技术组件 |
|------|------|----------|
| ODS | Kafka 原始数据接入 | Flink Kafka Source |
| DWD | 数据清洗、维度关联 | Flink DataStream |
| ADS | 5分钟滚动聚合 | Flink Window |
| Serving | 实时数据服务 | MySQL |

## 八、数据依赖关系

### 8.1 离线批处理依赖

```
Parquet文件
    ↓
ODS Loader (GreenOdsLoader / YellowOdsLoader)
    ↓
DWD Layer Builder (DwdLayerBuilder)
    ↓
DWS Layer Builder (DwsLayerBuilder)
    ↓
ADS Layer Builder (AdsLayerBuilder)
    ↓
MySQL ADS Tables
```

### 8.2 实时流处理依赖

```
Kafka (taxi_trip_green_raw)
    ↓
RealtimeOdsConsumer
    ↓
DataCleaner → BroadcastDimJoin
    ↓
RealtimeDwdProducer → Kafka (dwd_taxi_trip)
    ↓
Realtime5MinAggregation
    ↓
MySQLUpsertSink
    ↓
MySQL Realtime Tables
```

## 九、分层数据质量保障

### 9.1 ODS 层质量规则

| 规则 | 说明 | 处理方式 |
|------|------|----------|
| 源文件对账 | 源文件记录数 = ODS 记录数 | 记录差异，告警 |
| 格式校验 | JSON/Parquet 格式正确 | 错误数据入死信表 |
| 分区检查 | 分区目录是否存在 | 自动创建 |

### 9.2 DWD 层质量规则

| 规则 | 说明 | 处理方式 |
|------|------|----------|
| 空值检查 | 必填字段不能为空 | 过滤或填充默认值 |
| 范围检查 | 数值在合理范围内 | 过滤 |
| 关联检查 | 维度外键存在 | 过滤 |
| 一致性检查 | 时间字段逻辑正确 | 过滤 |

### 9.3 DWS 层质量规则

| 规则 | 说明 | 处理方式 |
|------|------|----------|
| 汇总校验 | 细粒度合 = 粗粒度 | 差异告警 |
| 趋势检查 | 与历史同期对比 | 异常告警 |
| 完整性检查 | 预期记录数 | 缺失告警 |

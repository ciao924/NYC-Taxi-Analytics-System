
# Batch Processor 模块分析报告

## 1. 模块概述

Batch Processor 是纽约市出租车数据分析系统的**离线批处理核心模块**，负责处理历史数据，构建完整的数仓分层结构（ODS → DWD → DWS → ADS）。该模块基于 Apache Spark 和 Apache Iceberg 技术栈，实现了高效、可靠的数据处理能力。

### 1.1 核心功能

| 功能模块 | 描述 | 状态 |
|---------|------|------|
| **ODS 数据加载** | 从 Parquet 文件加载原始数据到 Hive ODS 层 | ✅ |
| **DWD 明细层构建** | 数据清洗、标准化、维度关联、派生字段生成 | ✅ |
| **DWS 汇总层构建** | 多维度聚合指标计算，生成宽表 | ✅ |
| **ADS 应用层输出** | 将业务指标写入 MySQL，供前端 BI 使用 | ✅ |
| **维度表管理** | 维度数据的加载、同步和管理 | ✅ |
| **数据质量检测** | 多层级数据质量校验和报告生成 | ✅ |
| **缓存优化管理** | 智能缓存策略，提升性能 | ✅ |

### 1.2 处理流程

```
Parquet 文件 → ODS 加载 → DWD 清洗 → DWS 聚合 → ADS 输出
     ↓              ↓          ↓          ↓          ↓
  原始数据      元数据    标准化数据   汇总指标    MySQL
```

## 3. 核心实现分析

### 3.1 DWD 层构建 - DwdLayerBuilder.scala

#### 3.1.1 处理流程

**阶段划分**：
1. **数据源读取**：从 Hive ODS 读取绿/黄两种出租车数据
2. **合并统一**：将两种数据源合并，统一字段
3. **空值处理**：分层级处理 null 值（过滤/填充）
4. **值域过滤**：过滤异常值（距离、金额、人数等）
5. **维度关联**：广播关联维度表（支付、供应商、区域等）
6. **逻辑修复**：根据业务规则修复数据
7. **派生字段**：生成 trip_id、日期时间特征、行程类型等
8. **去重处理**：确保数据唯一性
9. **质量检测**：执行全面质量检查
10. **原子写入**：用 Iceberg 事务性覆盖写入

### 4.3 SmartCacheLite - 智能缓存

**职责**：轻量级缓存管理，自动追踪缓存状态

```scala
class SmartCacheLite {
  private val cachedDataFrames = ListBuffer[(DataFrame, String)]()
  
  def cache(df: DataFrame, name: String): DataFrame = {
    val cached = df.cache()
    cachedDataFrames += ((cached, name))
    cached
  }
  
  def uncacheAll(): Unit = {
    cachedDataFrames.foreach { case (df, name) =&gt;
      try df.unpersist() catch { case _: Exception =&gt; }
    }
    cachedDataFrames.clear()
  }
}
```

## 6. 性能优化策略

### 6.1 已实施的优化

| 优化项 | 说明 | 效果 |
|--------|------|------|
| **小表广播** | 维度表使用 Broadcast Join | 减少 Shuffle |
| **分区裁剪** | 按 year/month 分区过滤 | 减少数据扫描 |
| **列裁剪** | 只选择需要的列 | 减少内存占用 |
| **智能缓存** | 对中间结果缓存复用 | 减少重复计算 |
| **一次性聚合** | 单次扫描计算多个指标 | 减少 I/O |
| **Iceberg 格式** | 列式存储，支持 Schema 演进 | 查询性能提升 |

### 6.2 建议的优化方向

1. **AQE 启用**：Spark 3.x 自适应查询执行
2. **动态分区**：更细粒度的分区策略
3. **ZOrder 索引**：Iceberg ZOrder 优化多维查询
4. **增量处理**：支持增量更新而非全量覆盖

# 2026-07-15 前端UI设计规范化优化

## 问题描述

前后端UI设计不统一，各模块页面风格不一致，需要进行规范化优化以达到专业化、现代化、标准化的要求。

## 修复内容

### 1. 数据看板模块统一设计规范

**文件**: `taxi-analytics-frontend/src/views/dashboard/Index.vue`

定义统一的CSS类名和设计规范，作为其他模块的参考标准：

| 元素 | CSS类名 | 说明 |
|------|---------|------|
| 页面头部 | `dashboard-header` | flex布局，白色背景，圆角12px，阴影 |
| 头部左侧 | `header-left` | 包含标题和副标题 |
| 头部右侧 | `header-right` | 包含操作按钮 |
| 页面标题 | `page-title` | 24px，#1f2937 |
| 页面副标题 | `page-subtitle` | 14px，#6b7280 |
| KPI网格 | `kpi-grid` | grid布局，4列，20px间距 |
| KPI卡片 | `kpi-card` | 白色背景，24px内边距 |
| KPI指示器 | `kpi-indicator` | 8px宽，48px高，圆角4px |
| 图表区域 | `chart-section` | grid布局，2fr 1fr比例 |
| 图表卡片 | `chart-card` | 白色背景，圆角12px |
| 图表头部 | `chart-header` | 20px内边距，底部边框 |
| 图表主体 | `chart-body` | 20px内边距，320px高度 |

### 2. 实时监控模块UI风格统一

**文件**: `taxi-analytics-frontend/src/views/realtime/Index.vue`

- 采用 `dashboard-header` 替代原有 `page-header`
- 采用 `kpi-grid` + `kpi-card` 统一KPI展示布局
- 采用 `chart-section` + `chart-card` 统一图表容器样式
- 更新指示器颜色为统一的蓝色系
- 添加 `page-subtitle` 副标题样式

### 3. 数据质量模块UI风格统一

**文件**: `taxi-analytics-frontend/src/views/quality/Index.vue`

- 采用统一的设计规范头部
- KPI卡片使用相同的颜色指示器系统（good/warning/error）
- 图表区域布局与数据看板保持一致
- 统一进度条样式

### 4. 热力图模块UI风格统一

**文件**: `taxi-analytics-frontend/src/views/map/HeatmapView.vue`

- 采用 `dashboard-header` 统一页面头部
- 使用 `chart-card` 包装地图容器
- 统一间距和阴影效果

### 5. AI智能查询模块UI风格统一

**文件**: `taxi-analytics-frontend/src/views/ai/Index.vue`

- 采用 `dashboard-header` 替代原有的渐变头部
- 统一按钮样式
- 将紫色主题色 `#667eea` 统一改为蓝色主题色 `#3b82f6`
- 统一激活状态颜色为 `#eff6ff` + `#3b82f6`

# 出租车分析系统修复日志 - 2026年05月16日

## 修复记录

### 2026-05-16 - 历史会话图表渲染及chartConfig完整解析

**问题描述**：
1. 打开历史会话时，对话信息区域显示历史对话，但查询记录的可视化图没有渲染
2. 历史记录应该从MySQL获取，但不能正确获取chartConfig中的data字段

**根因分析**：

#### 问题1：前端selectSession函数未设置chartHeight
前端 `selectSession` 函数在加载历史消息时，没有为每条消息设置 `chartHeight`，导致图表容器高度为0，无法渲染。

#### 问题2：MySQL JSON类型字段返回格式不一致
后端 `AiMapper.getSessionMessages` 使用 `SELECT *` 查询，`chart_data` 字段是 MySQL JSON 类型。某些 JDBC 驱动可能将 JSON 类型自动解析为对象而不是字符串，导致后续 JSON 解析失败。

#### 问题3：chartConfig解析逻辑不完整
后端 `getSessionHistory` 方法在解析 `chartConfig` 时：
1. 只优先从 `chart_data` 解析，如果失败就不再尝试从 `content` 获取
2. 没有处理 `chart_data` 已经是 Map 类型的情况
3. 缺少调试日志，问题难以追踪

**修复方案**：

#### 1. 前端：修复selectSession函数，添加chartHeight计算
```javascript
// 前端 Index.vue - selectSession 函数
// 计算图表高度
const chartHeight = canShowChart({ chartConfig })
  ? calculateChartHeight(chartContainerWidth.value)
  : 0

return {
  messageId: msg.messageId || msg.message_id,
  role: msg.role,
  content: content,
  explanation: explanation,
  sql: msg.sql || msg.sql_text,
  chartConfig: chartConfig,
  chartHeight: chartHeight  // 新增：设置图表高度
}
```

#### 2. 后端：修改AiMapper查询，确保chart_data以字符串返回
```java
// AiMapper.java - getSessionMessages 方法
@Select("<script>SELECT id, session_id, message_id, role, content, sql_text, " +
        "IFNULL(chart_data, JSON_UNQUOTE('null')) AS chart_data, " +
        "execution_time_ms, row_count, create_time " +
        "FROM ai_message WHERE session_id = #{sessionId} ORDER BY create_time ASC</script>")
List<Map<String, Object>> getSessionMessages(@Param("sessionId") String sessionId);
```

            Map<String, Object> chartConfig = null;

                String chartDataStr = null;
                if (chartDataObj instanceof String) {
                    chartDataStr = (String) chartDataObj;
                } else if (chartDataObj instanceof Map) {
                    // 如果已经是 Map，直接使用

                        log.warn("从 chart_data 解析 chartConfig 失败: {}", e.getMessage());

                }

            }
        }
    }
    return messages;

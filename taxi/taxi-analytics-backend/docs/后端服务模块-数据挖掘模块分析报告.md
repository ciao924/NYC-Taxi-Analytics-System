# 数据挖掘模块分析报告

## 一、模块概述

### 1.1 模块定位

数据挖掘模块是 NYC Taxi Analytics System 的**智能分析核心**，通过丰富的统计和机器学习算法，从海量出租车运营数据中挖掘有价值的信息，为业务决策提供数据支持。

### 1.2 核心文件

| 文件 | 路径 | 职责 |
|------|------|------|
| **DataMiningUtils.java** | `modules/analysis/util/DataMiningUtils.java` | 数据挖掘算法工具类 |
| **AnalysisService.java** | `modules/analysis/service/AnalysisService.java` | 分析服务入口 |
| **AnalysisController.java** | `modules/analysis/controller/AnalysisController.java` | 分析接口控制器 |

### 1.3 技术特性

- **算法丰富**：涵盖时间序列、统计、异常检测、聚类四大类
- **性能优化**：增量计算、滑动窗口、多线程支持
- **业务导向**：直接对接业务场景，即插即用

## 二、核心算法详解

### 2.1 时间序列分析

#### 2.1.1 加权移动平均 (WMA)

**方法**: `calculateWeightedMovingAverage(List<Double> values, int windowSize)`

**原理**: 近期数据赋予更高权重，平滑数据噪声

```
WMA = (w1*v1 + w2*v2 + ... + wn*vn) / (w1 + w2 + ... + wn)
其中 wi = i (索引从1开始)
```

**应用场景**:
- 收入趋势平滑
- 订单量预测预处理
- 季节性数据去噪

#### 2.1.2 相对强弱指数 (RSI)

**方法**: `calculateRSI(List<Double> values, int period)`

**原理**: 衡量价格变化的幅度，识别超买超卖

```
RSI = 100 - (100 / (1 + RS))
RS = 平均涨幅 / 平均跌幅
```

**判断标准**:
- RSI > 70: 超买区域
- RSI < 30: 超卖区域
- RSI 持续高位: 强势上涨
- RSI 持续低位: 弱势下跌

#### 2.1.3 指数平滑预测

**方法**: `exponentialSmoothingForecast(List<Double> values, double alpha)`

**原理**: 通过指数衰减的权重组合历史数据

```
S_t = alpha * y_{t-1} + (1-alpha) * S_{t-1}
```

**参数说明**:
- `alpha (α)`: 平滑系数，范围 [0, 1]
- α 越大：对近期数据越敏感
- α 越小：历史数据影响越大

#### 2.1.4 霍尔特线性趋势预测

**方法**: `holtLinearForecast(List<Double> values, double alpha, double beta, int forecastPeriods)`

**原理**: 考虑趋势成分的二次指数平滑

```
Level:   L_t = alpha * y_t + (1-alpha) * (L_{t-1} + T_{t-1})
Trend:   T_t = beta * (L_t - L_{t-1}) + (1-beta) * T_{t-1}
Forecast: F_{t+h} = L_t + h * T_t
```

**应用场景**:
- 短期收入预测
- 订单量趋势预测
- 客流增长趋势分析

#### 2.1.5 时间序列分解

**方法**: `decomposeTimeSeries(List<Double> values)`

**原理**: 将时间序列分解为趋势、季节性、残差三部分

```
Y_t = T_t + S_t + R_t
- T_t: 趋势成分（长期变化）
- S_t: 季节性成分（周期性波动）
- R_t: 残差成分（随机噪声）
```

### 2.2 统计分析

#### 2.2.1 变异系数 (CV)

**方法**: `calculateCoefficientOfVariation(List<Double> values)`

**原理**: 标准差与均值的比值，用于比较不同数据集的离散程度

```
CV = (标准差 / 均值) * 100%
```

**判断标准**:
- CV < 15%: 低离散，数据稳定
- CV 15%~30%: 中等离散
- CV > 30%: 高离散，数据波动大

#### 2.2.2 Gini系数

**方法**: `calculateGiniCoefficient(List<Double> values)`

**原理**: 衡量分布不均匀程度，常用于收入差距分析

```
Gini = (2 * Σ(i * v_i)) / (n * Σv_i) - (n+1) / n
```

**判断标准**:
- Gini = 0: 完全均等
- Gini = 1: 完全不均等
- Gini < 0.2: 相对均等
- Gini > 0.4: 差距较大

#### 2.2.3 熵值

**方法**: `calculateEntropy(List<Double> values)`

**原理**: 衡量数据分布的不确定性

```
Entropy = -Σ(p_i * log2(p_i))
p_i = v_i / Σv_i
```

**应用场景**:
- 区域订单分布均匀度分析
- 支付方式多样性分析
- 客源地分布分析

#### 2.2.4 偏度与峰度

**方法**: `calculateSkewness(List<Double> values)`, `calculateKurtosis(List<Double> values)`

**原理**:
- 偏度：分布的不对称程度
- 峰度：分布的尖锐程度

**偏度判断**:
- Skewness = 0: 对称分布
- Skewness > 0: 右偏（正偏）
- Skewness < 0: 左偏（负偏）

**峰度判断**:
- Kurtosis = 0: 正态分布
- Kurtosis > 0: 尖峰分布（肥尾）
- Kurtosis < 0: 平峰分布（薄尾）

#### 2.2.5 线性回归

**方法**: `performLinearRegression(List<Double> xValues, List<Double> yValues)`

**返回**: `LinearRegressionResult(slope, intercept, rSquared)`

**原理**: 拟合线性关系 y = ax + b

```
a (slope) = Σ((x-mean_x)*(y-mean_y)) / Σ((x-mean_x)^2)
b (intercept) = mean_y - a * mean_x
R² = 1 - (SSR / SST)
```

**应用场景**:
- 距离与时长关系分析
- 订单量与收入相关性
- 人口密度与打车需求

---

### 2.3 异常检测

#### 2.3.1 Z-score 异常检测

**方法**: `detectAnomaliesByZScore(List<Double> values, double threshold)`

**原理**: 基于标准差的异常识别

```
Z_score = (v - mean) / std
|Z_score| > threshold → 异常
```

**严重程度分级**:

| Z分数范围 | 严重程度 | 说明 |
|-----------|----------|------|
| > 3σ | critical | 极端异常 |
| 2.5σ ~ 3σ | high | 高度异常 |
| 2σ ~ 2.5σ | medium | 中度异常 |

**应用场景**:
- 异常订单识别（金额异常高/低）
- 欺诈行为检测
- 数据录入错误识别

#### 2.3.2 IQR 异常检测

**方法**: `detectOutliersUsingIQR(List<Double> values)`

**原理**: 基于四分位距的离群值检测

```
Q1 = 25%分位数
Q3 = 75%分位数
IQR = Q3 - Q1
下界 = Q1 - 1.5 * IQR
上界 = Q3 + 1.5 * IQR
值 < 下界 或 值 > 上界 → 异常
```

**优势**: 不受极端值影响，对偏态分布友好

#### 2.3.3 趋势变化点检测

**方法**: `detectTrendChangePoints(List<Double> values, int windowSize)`

**原理**: 滑动窗口比较检测序列中的突变点

```
比较窗口[i, i+windowSize] 与 窗口[i+windowSize, i+2*windowSize] 的均值差异
差异超过阈值 → 变化点
```

### 2.4 聚类分析

#### 2.4.1 K-means 聚类

**方法**: `kMeansClustering(List<DataPoint<T>> points, int k, int maxIterations)`

**原理**: 将数据点划分为K个簇，使簇内平方和最小

**算法流程**:

```
1. 初始化: 随机选择K个数据点作为初始质心
2. 分配: 对每个数据点，计算到K个质心的距离，分配到最近簇
3. 更新: 重新计算每个簇的质心（均值）
4. 迭代: 重复步骤2-3，直到质心不再变化或达到最大迭代次数
```

**距离计算**: 欧氏距离

```
distance(p, q) = sqrt(Σ(p_i - q_i)²)
```

**应用场景**:
- 热点区域识别（基于经纬度聚类）
- 乘客类型划分（基于行为特征）
- 时段分类（基于订单分布）

**代码示例**:

```java
List<DataPoint<String>> points = new ArrayList<>();
for (TripRecord trip : tripRecords) {
    points.add(new DataPoint<>(
        trip.getLocationId(),
        new double[]{trip.getLatitude(), trip.getLongitude()}
    ));
}

List<Cluster<String>> clusters = 
    DataMiningUtils.kMeansClustering(points, 5, 100);

for (Cluster<String> cluster : clusters) {
    double[] centroid = cluster.getCentroid();
    log.info("热点区域: ({}, {}), 订单数={}", 
        centroid[0], centroid[1], cluster.getPoints().size());
}
```

### 2.5 模式识别

#### 2.5.1 周期性分析

**方法**: `analyzeSeasonality(List<Double> values)`

**原理**: 检测数据中的季节性模式

```java
public class SeasonalityResult {
    private boolean hasSeasonality;  // 是否有季节性
    private int period;              // 周期长度
    private double strength;          // 季节性强度 [0, 1]
}
```

**判断标准**: strength > 0.6 认为具有明显季节性

#### 2.5.2 互相关分析

**方法**: `calculateCrossCorrelation(List<Double> series1, List<Double> series2, int maxLag)`

**原理**: 分析两个序列在不同滞后下的相关性

```
CCF(lag) = Σ((x_{t} - mean_x) * (y_{t+lag} - mean_y)) / sqrt(Σ(x_t-mean_x)² * Σ(y_t-mean_y)²)
```

**应用场景**:
- 降雨量与订单量关系
- 节假日与收入增长关系
- 人口流动与打车需求

#### 2.5.3 趋势模式检测

**方法**: `detectPattern(List<Double> values)`

**返回**: 上升/下降/稳定

```
上升: 连续n个点呈递增趋势
下降: 连续n个点呈递减趋势
稳定: 变化幅度在阈值范围内
```

## 三、业务应用场景

### 3.1 应用场景矩阵

| 场景 | 核心算法 | 数据来源 | 业务价值 |
|------|----------|----------|----------|
| **收入预测** | 霍尔特线性预测 | ADS日报表 | 支持财务决策 |
| **异常订单检测** | Z-score、IQR | 实时流数据 | 欺诈防范 |
| **热点区域分析** | K-means聚类 | GPS位置数据 | 资源优化配置 |
| **季节性波动** | 周期分析 | 历史数据 | 运营策略调整 |
| **数据质量监控** | 统计指标 | 全链路数据 | 数据健康保障 |
| **收入差距分析** | Gini系数 | 订单数据 | 公平性评估 |
| **距离时长关系** | 线性回归 | 行程记录 | 定价策略优化 |

### 3.2 典型业务流程

#### 3.2.1 每日收入异常检测

```
1. Batch Processor 计算每日收入汇总 → 写入 ADS 层
2. AnalysisService 获取最近30天收入数据
3. DataMiningUtils.detectAnomaliesByZScore() 检测异常
4. 异常结果通过 WebSocket 推送至前端告警
5. 前端展示异常日期及严重程度
```

#### 3.2.2 热点区域发现

```
1. Streaming Processor 实时采集上下车位置
2. 定时任务每5分钟聚合一次位置数据
3. DataMiningUtils.kMeansClustering() 识别热点簇
4. 结果写入 MySQL热点表
5. 前端热力图展示Top N热点区域
```

#### 3.2.3 收入趋势预测

```
1. AnalysisService 获取历史收入序列
2. DataMiningUtils.holtLinearForecast() 预测未来7天
3. 结合季节性分析调整预测值
4. 结果缓存至 Redis
5. 前端仪表盘展示预测趋势
```

## 五、性能优化

### 5.1 算法复杂度

| 算法 | 时间复杂度 | 空间复杂度 | 适用规模 |
|------|------------|------------|----------|
| 加权移动平均 | O(n) | O(1) | 任意规模 |
| K-means聚类 | O(n*k*i) | O(n+k) | n < 100万 |
| Z-score检测 | O(n) | O(n) | 任意规模 |
| 线性回归 | O(n) | O(1) | 任意规模 |
| 霍尔特预测 | O(n) | O(1) | 任意规模 |

> n: 数据量, k: 簇数, i: 迭代次数

### 5.2 优化策略

1. **增量计算**: 滑动窗口采用 O(1) 更新
2. **采样分析**: 大规模数据先采样再分析
3. **并行处理**: 聚类分析支持多线程
4. **缓存复用**: 中间结果缓存避免重复计算

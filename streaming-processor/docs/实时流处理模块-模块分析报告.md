
# Streaming Processor 模块分析报告

## 1. 模块概述

Streaming Processor 是纽约市出租车数据分析系统的**实时流处理核心模块**，基于 Apache Flink 构建，负责消费 Kafka 中的实时数据，进行清洗、质量检测、指标计算，并输出到 HDFS 和 MySQL。

### 1.1 核心功能

| 功能 | 描述 | 状态 |
|------|------|------|
| **Kafka 消费** | 从 Kafka Topic 实时消费数据 | ✅ |
| **JSON 解析** | 解析 JSON 格式的出租车数据 | ✅ |
| **数据清洗** | 过滤无效数据，标准化字段 | ✅ |
| **质量检测** | 实时监控数据质量，生成报告 | ✅ |
| **告警机制** | 质量异常时触发告警 | ✅ |
| **ODS 落地** | 实时写入 HDFS ODS 层（ORC 格式） | ✅ |
| **指标计算** | 实时计算订单、热点、费用等指标 | ✅ |
| **多 Sink 输出** | 支持 HDFS、MySQL、死信队列 | ✅ |
| **状态管理** | Flink Checkpoint 和状态恢复 | ✅ |

## 3. 核心实现分析

### 3.1 RealtimeOdsJob - ODS 落地作业

**职责**：将 Kafka 中的原始数据清洗后写入 HDFS ODS 层，格式为 ORC，按 year/month 分区。

#### 3.1.1 处理流程

```scala
// 1. 初始化环境和配置
ConfigManager.init(args)
val env = StreamExecutionEnvironment.getExecutionEnvironment
configureEnvironment(env)

// 2. 从 Kafka 读取数据
val source = env.fromSource(
  KafkaSourceBuilder.build(ConfigManager.getKafkaConsumerGroupId),
  WatermarkStrategy.noWatermarks(),
  "Kafka Source"
)

// 3. JSON 解析
val jsonParser = new JsonParseFunction()
val parsedStream = source.process(jsonParser).name("JSON Parsing")

// 4. 解析错误 → 死信队列
val parsingErrors = parsedStream.getSideOutput(jsonParser.parsingErrors)
parsingErrors.addSink(DeadLetterSinkBuilder.build())

// 5. 数据清洗和时间验证
val odsStream = parsedStream
  .filter(_ != null)
  .process(new ProcessFunction[GreenTripRecord, GreenTripRecord] {
    override def processElement(record: GreenTripRecord, ctx: Context, out: Collector[GreenTripRecord]): Unit = {
      record.pickupDatetime match {
        case Some(ts) =>
          val year = Calendar.getInstance().get(Calendar.YEAR)
          if (year >= 2000 && year <= 2100) {
            out.collect(record)
          } else {
            ctx.output(invalidTimestampErrors, new ErrorRecordJava(...))
          }
        case None =>
          ctx.output(invalidTimestampErrors, new ErrorRecordJava(...))
      }
    }
  })

// 6. 按车型分流
val greenStream = odsStream.filter(_.taxiType.contains("green"))
val yellowStream = odsStream.filter(_.taxiType.contains("yellow"))

// 7. 写入 HDFS ORC
writeToHdfsWithOrcFileSink(greenStream, yellowStream)
```

#### 3.1.2 ORC 文件写入

```scala
private def writeToHdfsWithOrcFileSink(
  greenStream: DataStream[GreenTripRecord],
  yellowStream: DataStream[GreenTripRecord]
): Unit = {
  
  // ORC 配置：SNAPPY 压缩
  val hadoopConf = new Configuration()
  hadoopConf.set("orc.compress", "SNAPPY")
  hadoopConf.set("orc.stripe.size", "268435456")  // 256MB
  hadoopConf.set("orc.row.index.stride", "10000")
  
  // ORC Writer Factory
  val vectorizer = new GreenTripRecordVectorizer()
  val writerFactory = new OrcBulkWriterFactory[GreenTripRecord](vectorizer, hadoopConf)
  
  // 文件 Sink 配置
  val greenFileSink = FileSink
    .forBulkFormat(new Path(greenTripPath), writerFactory)
    .withBucketAssigner(new GreenTripRecordBucketAssigner())  // 按时间分桶
    .withRollingPolicy(OnCheckpointRollingPolicy.build())  // Checkpoint 时滚动
    .withOutputFileConfig(
      OutputFileConfig.builder()
        .withPartPrefix("green")
        .withPartSuffix(".orc")
        .build()
    )
    .build()
  
  greenStream.sinkTo(greenFileSink)
}
```

#### 3.1.3 分桶策略

```scala
class GreenTripRecordBucketAssigner extends BucketAssigner[GreenTripRecord, String] {
  override def getBucketId(
    element: GreenTripRecord,
    context: BucketAssigner.Context
  ): String = {
    element.pickupDatetime match {
      case Some(ts) =>
        val cal = Calendar.getInstance()
        cal.setTimeInMillis(ts.getTime)
        val year = cal.get(Calendar.YEAR)
        val month = f"${cal.get(Calendar.MONTH) + 1}%02d"
        s"year=$year/month=$month"  // Hive 分区格式
      case None =>
        "year=unknown/month=unknown"
    }
  }
}
```

### 3.2 RealtimeMetricsJob - 指标计算作业

**职责**：实时计算订单指标、热点区域、费用构成等业务指标，写入 MySQL。

#### 3.2.1 核心指标计算

**1. 订单指标聚合**

```scala
// 按 (pickupZone, paymentType) 分组，5 分钟滚动窗口
val orderMetrics = tripStream
  .keyBy(new KeySelector[Trip, (String, String)] {
    override def getKey(t: Trip): (String, String) = (t.pickupZone, t.paymentType)
  })
  .window(TumblingProcessingTimeWindows.of(Time.minutes(5)))
  .aggregate(
    new OrderMetricsAggregateFunction(),  // 增量聚合
    new OrderMetricsWindowFunction()       // 窗口函数
  )

// 聚合函数：计数 + 求和
class OrderMetricsAggregateFunction extends AggregateFunction[Trip, (Long, Double), (Long, Double)] {
  override def createAccumulator(): (Long, Double) = (0L, 0.0)
  override def add(t: Trip, acc: (Long, Double)): (Long, Double) =
    (acc._1 + 1, acc._2 + t.fareAmount)
  override def getResult(acc: (Long, Double)): (Long, Double) = acc
  override def merge(a: (Long, Double), b: (Long, Double)): (Long, Double) =
    (a._1 + b._1, a._2 + b._2)
}

// 窗口函数：输出 OrderMetric
class OrderMetricsWindowFunction extends ProcessWindowFunction[...] {
  override def process(...): Unit = {
    val (cnt, sum) = elements.iterator().next()
    out.collect(OrderMetric(
      windowStart = context.window.getStart,
      windowEnd = context.window.getEnd,
      city = "NYC",
      orderCount = cnt,
      totalFare = sum,
      avgFare = if (cnt > 0) sum / cnt else 0.0
    ))
  }
}
```

**2. 热点 TOPN**

```scala
// 第一步：按区域聚合
val zoneCounts = tripStream
  .keyBy(_.pickupZone)
  .window(TumblingProcessingTimeWindows.of(Time.minutes(5)))
  .aggregate(new ZoneCountAggregateFunction(), new ZoneCountWindowFunction())

// 第二步：全局 TOPN（使用 KeyedState）
val hotspotTopN = zoneCounts
  .keyBy(_._1)  // 按窗口结束时间分组
  .process(new HotspotTopNProcessFunction(10))

class HotspotTopNProcessFunction(n: Int) extends KeyedProcessFunction[Long, ...] {
  private var zoneCounts: ListState[(String, Long)] = _
  
  override def open(...): Unit = {
    val stateDesc = new ListStateDescriptor[(String, Long)]("zoneCounts", ...)
    zoneCounts = getRuntimeContext.getListState(stateDesc)
  }
  
  override def processElement(value: ..., ctx: Context, out: Collector[HotspotResult]): Unit = {
    zoneCounts.add((value._2, value._3))  // (zone, count)
    ctx.timerService().registerProcessingTimeTimer(value._1 + 1)  // 窗口结束后触发
  }
  
  override def onTimer(timestamp: Long, ctx: OnTimerContext, out: Collector[HotspotResult]): Unit = {
    val list = scala.collection.mutable.ListBuffer[(String, Long)]()
    val iter = zoneCounts.get().iterator()
    while (iter.hasNext) list += iter.next()
    
    // 排序取 TOPN
    val sorted = list.sortBy(-_._2).take(n)
    sorted.zipWithIndex.foreach { case ((zone, cnt), idx) =>
      out.collect(HotspotResult(
        windowStart = ctx.getCurrentKey - Time.minutes(5).toMilliseconds,
        windowEnd = ctx.getCurrentKey,
        zone = zone,
        cnt = cnt,
        rank = idx + 1
      ))
    }
    
    zoneCounts.clear()
  }
}
```

### 3.3 质量检测与告警

#### 3.3.1 质量事件流

```scala
if (ConfigManager.isDataQualityEnabled) {
  // 转换各类事件为统一的 QualityEvent
  val normalStream = validatedStream.map(_ => NormalEvent(): QualityEvent)
  val parseErrorStream = parsingErrors.map(_ => ParseErrorEvent(): QualityEvent)
  val cleanErrorStream = cleaningErrors.map(_ => CleanErrorEvent(): QualityEvent)
  val timestampErrorStream = validatedStream.getSideOutput(invalidTimestampErrors)
    .map(_ => TimestampErrorEvent(): QualityEvent)
  
  // 合并所有事件流
  val allEvents = normalStream.union(parseErrorStream, cleanErrorStream, timestampErrorStream)
    .map(event => ("quality_key", event))
  
  // 5 分钟聚合质量指标
  val qualityResults = allEvents
    .keyBy(_._1)
    .window(TumblingProcessingTimeWindows.of(Time.minutes(5)))
    .aggregate(new QualityAggregateFunction(), new QualityWindowFunction())
  
  // 输出质量报告
  qualityResults.addSink(QualitySinkBuilder.build())
  
  // 告警检测
  val alertStream = qualityResults
    .keyBy(_ => "alert_key")
    .process(new AlertDetector())
  
  alertStream.addSink(AlertSinkBuilder.build())
}
```

#### 3.3.2 告警检测器

```scala
class AlertDetector extends KeyedProcessFunction[String, QualityResult, Alert] {
  override def processElement(result: QualityResult, ctx: Context, out: Collector[Alert]): Unit = {
    // 告警规则：错误率超过 5% 触发告警
    val errorRate = if (result.totalRecords > 0) {
      (result.parseError + result.cleanError + result.timestampError).toDouble / result.totalRecords
    } else 0.0
    
    if (errorRate > 0.05) {
      out.collect(Alert(
        timestamp = System.currentTimeMillis(),
        alertType = "HIGH_ERROR_RATE",
        message = s"Error rate is ${errorRate * 100}%, exceeds threshold 5%",
        severity = "WARNING",
        details = result
      ))
    }
  }
}
```

### 3.4 Flink 环境配置

```scala
private def configureEnvironment(env: StreamExecutionEnvironment): Unit = {
  // 1. 并行度
  env.setParallelism(ConfigManager.getFlinkParallelism)
  
  // 2. 状态后端：RocksDB (生产级)
  env.setStateBackend(new EmbeddedRocksDBStateBackend())
  env.getCheckpointConfig.setCheckpointStorage(checkpointPath)
  
  // 3. Checkpoint 配置：EXACTLY_ONCE，2 分钟间隔
  env.enableCheckpointing(120000, CheckpointingMode.EXACTLY_ONCE)
  env.getCheckpointConfig.setCheckpointTimeout(10 * 60 * 1000)  // 10 分钟超时
  env.getCheckpointConfig.setMinPauseBetweenCheckpoints(60000)    // 最小间隔 60 秒
  env.getCheckpointConfig.setMaxConcurrentCheckpoints(1)          // 最多 1 个并发
  env.getCheckpointConfig.setTolerableCheckpointFailureNumber(3)  // 容忍 3 次失败
  
  // 4. 外部化 Checkpoint：取消时保留
  env.getCheckpointConfig.setExternalizedCheckpointCleanup(
    ExternalizedCheckpointCleanup.RETAIN_ON_CANCELLATION
  )
  
  // 5. 重启策略：固定延迟，最多 3 次，间隔 10 秒
  env.setRestartStrategy(
    RestartStrategies.fixedDelayRestart(3, Time.seconds(10))
  )
}
```

## 6. 监控与告警

### 6.1 关键监控指标

| 指标 | 说明 | 告警阈值 |
|------|------|---------|
| Kafka Lag | Topic 消息堆积 | &gt; 10000 条 |
| Checkpoint 失败率 | Checkpoint 失败比例 | &gt; 20% |
| 数据质量得分 | 质量检测得分 | &lt; 90 分 |
| 错误率 | 解析/清洗错误率 | &gt; 5% |
| 作业重启次数 | 自动重启次数 | &gt; 3 次/小时 |

### 6.2 死信队列监控

```sql
-- 查询死信队列中的错误记录
SELECT error_type, COUNT(*) as error_count, MIN(timestamp) as first_error, MAX(timestamp) as last_error
FROM dead_letter_queue
WHERE timestamp &gt;= DATE_SUB(NOW(), INTERVAL 1 HOUR)
GROUP BY error_type
ORDER BY error_count DESC;
```

# 实时数据质量检测：纽约出租车数据

```markdown
根据你提供的 `nyc_taxi_quality` 库表结构，我将设计一个完整的实时数据质量检测模块，集成到现有的 `RealtimeMetricsJob` 中。该模块将：

- 每 **5 分钟** 统计一次各错误类型的数量与比率
- 将统计结果写入 `data_quality_daily` 表（按天 + 检查类型存储，每个窗口单独一条记录，通过 `detail_json` 保留窗口时间）
- 根据 `quality_alert_config` 配置触发告警，写入 `quality_alert_history`
- 完全基于现有代码的侧输出流，不侵入原有业务逻辑

## 一、新增模型类（case class）

```scala
// com.taxi.realtime.quality.QualityMetric
package com.taxi.realtime.quality

import java.sql.Timestamp

case class QualityMetric(
    checkDate: java.sql.Date,          // data_quality_daily.check_date
    tableName: String,                 // 此处固定为 "realtime_trip"
    checkType: String,                 // 如 "json_parse_error_rate"
    checkStatus: String,               // "PASS" 或 "FAIL"
    expectedValue: Option[BigDecimal], // 期望值（阈值）
    actualValue: Option[BigDecimal],   // 实际值（比率或计数）
    deviationRate: Option[BigDecimal], // 偏差率
    detailJson: Option[String]         // 包含窗口起止、各错误计数等
)

// 窗口聚合中间结果
case class QualityWindowResult(
    windowStart: Long,
    windowEnd: Long,
    totalRecords: Long,
    parseErrors: Long,
    cleanErrors: Long,
    timestampErrors: Long
) {
  def validRecords: Long = totalRecords - parseErrors - cleanErrors - timestampErrors
  def parseErrorRate: BigDecimal = if (totalRecords == 0) 0 else BigDecimal(parseErrors) / BigDecimal(totalRecords)
  def cleanErrorRate: BigDecimal = if (totalRecords == 0) 0 else BigDecimal(cleanErrors) / BigDecimal(totalRecords)
  def overallQualityRate: BigDecimal = if (totalRecords == 0) 100 else BigDecimal(validRecords) / BigDecimal(totalRecords) * 100
}
```

## 二、实现质量统计窗口

在 `RealtimeMetricsJob` 中，我们已经有三类错误流：

- `parsingErrors` \(JSON 解析失败\)

- `cleaningErrors` \(数据质量规则失败，如距离、乘客数等\)

- `invalidTimestampErrors`（需要从 `RealtimeOdsJob` 同步过来，或直接在 `RealtimeMetricsJob` 中添加相同的时间戳校验）

**建议**：在 `RealtimeMetricsJob` 中也加入时间戳校验（与 ODS 作业一致），以便统一统计。

### 1\. 在 `RealtimeMetricsJob` 中增加校验侧输出

```scala
// 在 cleaned 流之后，添加与 ODS 相同的时间戳校验
val invalidTimestampErrors = new OutputTag[ErrorRecordJava]("invalid-timestamp") {}

val validatedStream = cleaned
  .process(new ProcessFunction[GreenTripRecord, GreenTripRecord] {
    override def processElement(record: GreenTripRecord, ctx: ProcessFunction[GreenTripRecord, GreenTripRecord]#Context, out: Collector[GreenTripRecord]): Unit = {
      record.pickupDatetime match {
        case Some(ts) =>
          val cal = Calendar.getInstance()
          cal.setTimeInMillis(ts.getTime)
          val year = cal.get(Calendar.YEAR)
          if (year >= 2000 && year <= 2100) out.collect(record)
          else ctx.output(invalidTimestampErrors, new ErrorRecordJava(record.toString, "Invalid year: " + year))
        case None =>
          ctx.output(invalidTimestampErrors, new ErrorRecordJava(record.toString, "Missing pickupDatetime"))
      }
    }
  })
  .name("Timestamp Validation")
  .uid("timestamp-validation")

// 将校验后的流作为后续转换的输入（替代原来的 cleaned）
val tripStream = validatedStream.map(...)

### 2\. 合并主流与三个错误流，进行窗口统计

import org.apache.flink.streaming.api.scala._
import org.apache.flink.streaming.api.windowing.assigners.TumblingProcessingTimeWindows
import org.apache.flink.streaming.api.windowing.time.Time
import org.apache.flink.streaming.api.functions.ProcessFunction
import org.apache.flink.util.Collector

// 定义一个统一的数据类型，用于标记记录类型
sealed trait QualityEvent
case class NormalEvent() extends QualityEvent
case class ParseErrorEvent() extends QualityEvent
case class CleanErrorEvent() extends QualityEvent
case class TimestampErrorEvent() extends QualityEvent

// 将四个流转换成统一的 QualityEvent，并携带一个虚拟 key 用于窗口
val normalStream = validatedStream.map(_ => NormalEvent())
val parseErrorStream = parsingErrors.map(_ => ParseErrorEvent())
val cleanErrorStream = cleaningErrors.map(_ => CleanErrorEvent())
val timestampErrorStream = invalidTimestampErrors.map(_ => TimestampErrorEvent())

val allEvents = normalStream.union(parseErrorStream, cleanErrorStream, timestampErrorStream)
  .map(event => ("quality_key", event))
  .keyBy(_._1)
  .window(TumblingProcessingTimeWindows.of(Time.minutes(5)))
  .aggregate(
    new QualityAggregateFunction,
    new QualityWindowFunction
  )

### 3\. 窗口聚合函数实现

```scala
import org.apache.flink.api.common.functions.AggregateFunction

class QualityAggregateFunction extends AggregateFunction[(String, QualityEvent), QualityWindowResult, QualityWindowResult] {
  override def createAccumulator(): QualityWindowResult = QualityWindowResult(0L, 0L, 0L, 0L, 0L, 0L)

  override def add(value: (String, QualityEvent), acc: QualityWindowResult): QualityWindowResult = {
    val (windowStart, windowEnd, total, parseErr, cleanErr, tsErr) = (acc.windowStart, acc.windowEnd, acc.totalRecords, acc.parseErrors, acc.cleanErrors, acc.timestampErrors)
    value._2 match {
      case NormalEvent() => acc.copy(totalRecords = total + 1)
      case ParseErrorEvent() => acc.copy(totalRecords = total + 1, parseErrors = parseErr + 1)
      case CleanErrorEvent() => acc.copy(totalRecords = total + 1, cleanErrors = cleanErr + 1)
      case TimestampErrorEvent() => acc.copy(totalRecords = total + 1, timestampErrors = tsErr + 1)
    }
  }

  override def getResult(acc: QualityWindowResult): QualityWindowResult = acc

  override def merge(a: QualityWindowResult, b: QualityWindowResult): QualityWindowResult = {
    QualityWindowResult(
      math.min(a.windowStart, b.windowStart),
      math.max(a.windowEnd, b.windowEnd),
      a.totalRecords + b.totalRecords,
      a.parseErrors + b.parseErrors,
      a.cleanErrors + b.cleanErrors,
      a.timestampErrors + b.timestampErrors
    )
  }
}

class QualityWindowFunction extends ProcessWindowFunction[QualityWindowResult, QualityWindowResult, String, TimeWindow] {
  override def process(key: String, context: Context, elements: Iterable[QualityWindowResult], out: Collector[QualityWindowResult]): Unit = {
    val result = elements.iterator.next()
    out.collect(result.copy(windowStart = context.window.getStart, windowEnd = context.window.getEnd))
  }
}
```

## 三、将窗口结果写入 `data\_quality\_daily`

定义一个 `QualitySinkFunction`，将 `QualityWindowResult` 转换为 `data\_quality\_daily` 表的记录，并写入 MySQL。同时在此处触发告警检测。

```scala
// com.taxi.realtime.sink.QualitySinkBuilder
package com.taxi.realtime.sink

import com.taxi.realtime.config.ConfigManager
import com.taxi.realtime.quality.{QualityMetric, QualityWindowResult}
import org.apache.flink.connector.jdbc.{JdbcConnectionOptions, JdbcExecutionOptions, JdbcSink}
import org.apache.flink.streaming.api.functions.sink.SinkFunction
import java.sql.{Date, PreparedStatement}
import java.time.Instant
import com.fasterxml.jackson.databind.ObjectMapper
import com.fasterxml.jackson.module.scala.DefaultScalaModule

object QualitySinkBuilder {
  private val mapper = new ObjectMapper().registerModule(DefaultScalaModule)

  def build(): SinkFunction[QualityWindowResult] = {
    val insertSql =
      """
        |INSERT INTO data_quality_daily
        |(check_date, table_name, check_type, check_status, expected_value, actual_value, deviation_rate, detail_json)
        |VALUES (?, ?, ?, ?, ?, ?, ?, ?)
      """.stripMargin

    JdbcSink.sink(
      insertSql,
      (ps: PreparedStatement, result: QualityWindowResult) => {
        val checkDate = new Date(Instant.now.toEpochMilli)  // 当天日期
        // 写入三条质量指标：解析错误率、清洗错误率、总体质量率
        val metrics = Seq(
          ("json_parse_error_rate", result.parseErrorRate, 5.0),     // 期望值 5%
          ("data_clean_error_rate", result.cleanErrorRate, 10.0),
          ("overall_quality_rate", result.overallQualityRate, 95.0)
        )
        metrics.foreach { case (checkType, actualValue, expectedValue) =>
          val status = if (actualValue <= expectedValue) "PASS" else "FAIL"  // 对于错误率，实际<=期望为通过；对于质量率，实际>=期望为通过
          val deviation = if (expectedValue != 0) (actualValue - expectedValue) / expectedValue else 0
          val detail = mapper.writeValueAsString(Map(
            "windowStart" -> result.windowStart,
            "windowEnd" -> result.windowEnd,
            "totalRecords" -> result.totalRecords,
            "parseErrors" -> result.parseErrors,
            "cleanErrors" -> result.cleanErrors,
            "timestampErrors" -> result.timestampErrors
          ))
          ps.setDate(1, checkDate)
          ps.setString(2, "realtime_trip")
          ps.setString(3, checkType)
          ps.setString(4, status)
          ps.setBigDecimal(5, BigDecimal.decimal(expectedValue).bigDecimal)
          ps.setBigDecimal(6, actualValue.bigDecimal)
          ps.setBigDecimal(7, deviation.bigDecimal)
          ps.setString(8, detail)
          ps.addBatch()
        }
        ps.executeBatch()
      },
      JdbcExecutionOptions.builder().withBatchSize(3).build(),
      new JdbcConnectionOptions.JdbcConnectionOptionsBuilder()
        .withUrl(ConfigManager.getMysqlUrl)
        .withUsername(ConfigManager.getMysqlUsername)
        .withPassword(ConfigManager.getMysqlPassword)
        .build()
    )
  }
}
```

## 四、告警检测与写入 `quality\_alert\_history`

在窗口结果输出前，增加一个 `AlertDetectionFunction`，查询 `quality\_alert\_config` 配置（可缓存，定期刷新），判断是否需要告警。

```scala
// com.taxi.realtime.quality.AlertDetector
class AlertDetector extends KeyedProcessFunction[String, QualityWindowResult, AlertRecord] {
  private var alertConfigs: ListState[AlertConfig] = _

  override def open(parameters: Configuration): Unit = {
    // 初始化时加载一次配置，并设置定时器定期刷新
    loadAlertConfigs()
    val timerService = getRuntimeContext.getTimerService
    timerService.registerProcessingTimeTimer(timerService.currentProcessingTime() + 60000) // 每分钟刷新
  }

  override def processElement(value: QualityWindowResult, ctx: KeyedProcessFunction[String, QualityWindowResult, AlertRecord]#Context, out: Collector[AlertRecord]): Unit = {
    import scala.collection.JavaConverters._
    val configs = alertConfigs.get().asScala.toList
    // 检查每条告警规则
    configs.foreach { config =>
      val actual = config.checkType match {
        case "json_parse_error_rate" => value.parseErrorRate
        case "data_clean_error_rate" => value.cleanErrorRate
        case "overall_quality_rate" => value.overallQualityRate
        case _ => return
      }
      val threshold = if (config.thresholdType == "WARNING") config.warningThreshold else config.criticalThreshold
      val isTriggered = config.checkType match {
        case "overall_quality_rate" => actual < threshold   // 质量率低于阈值告警
        case _ => actual > threshold                         // 错误率高于阈值告警
      }
      if (isTriggered) {
        out.collect(AlertRecord(
          alertConfigId = config.id,
          alertLevel = config.thresholdType,
          alertContent = s"${config.checkType} 实际值 ${actual} 超过阈值 ${threshold}",
          checkDate = new java.sql.Date(System.currentTimeMillis()),
          tableName = "realtime_trip",
          checkType = config.checkType,
          actualValue = actual,
          thresholdValue = threshold,
          isResolved = false,
          resolveTime = None,
          createTime = new Timestamp(System.currentTimeMillis())
        ))
      }
    }
  }

  override def onTimer(timestamp: Long, ctx: KeyedProcessFunction[String, QualityWindowResult, AlertRecord]#OnTimerContext, out: Collector[AlertRecord]): Unit = {
    // 定期刷新告警配置
    loadAlertConfigs()

## 七、注意事项

1. **时区**：`check\_date` 使用系统当前日期，建议统一使用 `Asia/Shanghai` 时区。

2. **性能**：每 5 分钟每个并行度都会输出窗口结果，但最终写入 MySQL 时会进行批量写入，压力很小。

3. **幂等性**：由于 `data\_quality\_daily` 没有唯一键，相同窗口可能会重复写入（作业重启后）。可以在写入前检查是否已存在相同 `check\_date`、`check\_type` 和 `detail\_json` 中的窗口时间，或使用 `INSERT IGNORE`。简单起见，可接受少量重复（每天最多 288 个窗口，重复几条无影响）。

4. **告警去重**：可以在 `AlertDetector` 中维护已告警的窗口 ID 状态，避免同一窗口重复告警。

5. **配置表动态更新**：`quality\_alert\_config` 的更新可以通过广播流实现，这里给出简化版（定期查询 MySQL）。
